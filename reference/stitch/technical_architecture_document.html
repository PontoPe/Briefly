# Technical Architecture: Editorial Marketplace ## 1. Frontend Stack - **Framework:** Next.js (App Router) - **Language:** TypeScript - **Styling:** Tailwind CSS (mapped to Design System tokens) - **Server State:** React Query (TanStack Query) - **Client State:** Zustand - **Real-Time Collaboration:** Liveblocks (for annotations and multiplayer presence) ## 2. Backend & Data - **Language/Framework:** Go (long-term performance) or NestJS (speed to market) - **API:** GraphQL (Apollo/Mercurius) for complex nested relationships - **Core Database:** PostgreSQL (managed via Prisma if NestJS) - **Caching & Presence:** Redis (session management, rate limiting, live indicators) - **Search Engine:** Meilisearch or Typesense (faceted filtering for granular search) ## 3. Infrastructure & DevOps - **Hosting (Frontend):** Vercel - **Hosting (Backend):** AWS ECS (Fargate) or Railway - **CI/CD:** GitHub Actions - **Containerization:** Docker - **Monitoring:** Sentry (errors), PostHog/Mixpanel (product analytics), Datadog (infrastructure) ## 4. Specialized Services - **Payments & Escrow:** Stripe Connect (Custom mode for milestone-based releases and split payments) - **Storage:** AWS S3 + CloudFront - **Asset Processing:** imgproxy or Thumbor (dynamic thumbnail generation) - **Auth & Security:** Auth0 or Clerk (MFA, JWT rotation, session management) - **AI Integration:** Anthropic (Claude) or OpenAI for "AI Brief Assist" features ## 5. Security Protocols - JWT with short expiration + rotation - AES-256 encryption at rest for sensitive data (Contracts, NDAs) - Webhook-driven IP transfer triggers - Malware scanning for file uploads via S3 integration
